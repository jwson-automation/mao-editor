# Contributing to MAO

Thank you for your interest in contributing to MAO!

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/mao.git`
3. Follow the [SETUP.md](SETUP.md) guide to set up your development environment
4. Create a new branch: `git checkout -b feature/your-feature-name`

## Development Workflow

1. Make your changes
2. Test your changes locally with `npm run dev`
3. Commit your changes with a clear commit message
4. Push to your fork: `git push origin feature/your-feature-name`
5. Create a Pull Request

## Commit Message Guidelines

- Use clear, descriptive commit messages
- Start with a verb (Add, Fix, Update, Remove, etc.)
- Keep the first line under 50 characters
- Example: `Add toggle visibility button to toolbar`

## Code Style

- Follow the existing code style
- Use TypeScript for type safety
- Use Vue 3 Composition API
- Keep components small and focused

## Testing

Before submitting a PR:
- Test all features locally
- Ensure the build completes without errors: `npm run build`
- Check that no console errors appear

## Pull Request Process

1. Update documentation if needed
2. Ensure your PR description clearly describes the changes
3. Link any related issues in the PR description
4. Wait for review and address any feedback

## Questions?

Feel free to open an issue for any questions or discussions!
